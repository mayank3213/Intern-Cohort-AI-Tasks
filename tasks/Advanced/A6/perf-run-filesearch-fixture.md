### What the profile showed

The `array_merge` call at `fixture/FileScanner.php:28` sits inside the recursive `fileSearch` method. Every time a subdirectory is processed, the returned array of child paths is merged into `$files` via `array_merge`, which **copies every element** into a new backing array. With a tree of depth 5 and 1,365 directories, the same path strings are copied repeatedly as results bubble up through each level of recursion. On a 13,650-file workload, this produces approximately O(n ├Ś depth) = ~68,250 redundant copy operations. Memory allocation and GC pressure from these short-lived intermediate arrays dominate the runtime, far exceeding the underlying directory I/O cost once the OS page cache is warm.

## Bottleneck Analysis

### Root cause

`FileScanner::fileSearch` uses `$files = array_merge($files, self::fileSearch($path, $extension))` at `source: fixture/FileScanner.php:28`. PHP's `array_merge` always allocates a new array and copies all elements from both operands; it cannot reuse existing backing storage. Because the call is recursive, each directory's results are re-copied at every ancestor level. The total copy work scales as O(n ├Ś d) where n = total file count and d = tree depth, making this an allocation_churn bottleneck with a secondary O(n┬▓) worst-case when many files bubble through many levels.

### Candidates considered

| rank | hotspot | type | fix_score | selected |
|---:|---|---|---:|---|
| 1 | `array_merge` in recursive accumulation loop | `allocation_churn` | 18.75 | yes |
| 2 | `opendir`/`readdir` per-directory syscalls | `io_blocking` | 3.0 | no ŌĆö dominated by cache hits; fix requires `RecursiveDirectoryIterator` rewrite, violates safety budget |
| 3 | `is_dir` + string concat per entry | `cpu_bound` | 1.2 | no ŌĆö too minor |

fix_score for rank-1: profile_dominance(5) ├Ś measurement_clarity(5) / behavior_risk(1.33) = **18.75**

### Fix rationale

Replace the `array_merge` call with a `foreach` loop that appends child paths one-by-one using `$files[] = $childFile`. This changes zero bytes of externally observable behaviour (same files found, same count, same path strings), modifies exactly one line of logic in a single file, and eliminates all intermediate array allocations. The fix is 1 file, +2 ŌłÆ1 lines, well inside the safety budget.

Instead of allocating a new array and copying all elements on every return, child paths are appended directly to the already-allocated `$files` array using `$files[] = $childFile`.
